<a name="Introduction"></a>
## Introduction
X-ray cameras are used in the field of airport operations and road safety, for example at customs. With the rapid development of society, the speed and accuracy of safety inspection are becoming more and more important. But traditional X-ray equipment is laborious, inefficient, and lacks a lengthy process, which requires mapping and lengthy inspections. These issues require automated tracking systems to evaluate X-ray security tests and improve the response time of human operators. 

A high-quality dataset is the cornerstone of a deep learning model with excellent performance.   A smaller number of datasets containing enough X-ray images with annotation has always a major obstacle to object detection in the actual security inspection work.   Thus, we construct a large-scale X-ray image dataset, i.e. CGDXray, to promote the solution of the above problem for more researchers to study.   We will introduce the detailed information about this benchmark.


<a name="CGDXray"></a>
## CGDXray for Xray Object Detection
We mainly collected X-ray images generated by security inspection machines from real scenarios such as subways, high-speed railways, etc. Due to the scarcity of extremely dangerous contraband goods in daily life, we also hired some volunteers to keep putting packages containing different contraband goods on the conveyor belt of the security machine in our laboratory to collect corresponding X-ray images. Six security machines produced by different manufactures were used in the data collection process, so there are six kinds of X-ray images with different styles. The X-ray images generated by different machines are different in color, size, etc. Figure below displays X-ray images from six different domains.

* **Notice** that all knives, glass bottles, drink bottles and tools are marked with fine-grained labels, while other objects are marked with coarse labels.
